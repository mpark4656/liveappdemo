Flex-Basis, Grow, and Shrink

Flex-basis is the initial size that our items need to be added as to the flex box. It overrides the width of the item. It may be a width or height depending on the main axis (row vs. column).

As shown below, in the row flexbox, setting flex-basis to 200px forces the first box to have the 200px width, regardless of its original width. min-width and max-width still trump the flex-basis size.

flex-grow is assigned a unit-less value and the value represents the proportion of the space that the item must take up given there is enough space. For example, an item with flex-grow of 2 will take up twice as much space as an item with flex-grow of 1, but only if there is enough space (The default width of all items will not change, meaning the size of other items won't shrink to support the defined proportion).

Flex Shorthand

Flex-basis, grow, and shrink are common that there is a shorthand for this.

  • One value, unit-less number: flex-grow
    flex: 2;
  • One value with unit: flex-basis
    flex: 10px;
  • Two values: flex-grow | flex-basis
    flex: 2 30px;
  • Two values: flex-grow | flex-shrink
    flex: 2 2;
  • Three values: flex-grow | flex-shrink | flex-basis
    flex: 2 2 10%;
